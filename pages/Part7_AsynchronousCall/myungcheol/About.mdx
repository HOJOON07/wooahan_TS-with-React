### 불확실한 응답에 대한 unknown (p208)

- API 응답 중 불확실한 값이 있고 로직에 영향을 주지 않는 경우에는 `unknown` 타입을 사용한다.
- 불확실한데 예제처럼 타입단언해서 사용하면 타입안정성이 떨어지지 않을까?

## 뷰모델 (p211)

- 서버응답을 뷰에 필요한 형태로 재정의하여 사용한다.
- 예제에서는 `fetchJobList`라는 API 호출함수 안에서 뷰모델 인스턴스를 생성하여 반환하도록 되어 있습니다.
- 같은 API를 호출하더라도, 나타나는 페이지나 컴포넌트에 따라서 뷰모델의 구성이 달라질 수 있다고 생각되는데요.
- 저의 경우는 API 호출 함수의 반환은 서버응답으로 고유하도록 유지하고, 필요한 곳에서 다시 처리할 수 있도록 재사용성을 고려하였었는데요.
- 이에 대해서 어떻게 생각하시나요?
