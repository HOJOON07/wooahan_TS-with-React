## About

- 요즘 context API와 SWR, React Query와 같은 데이터 페칭 라이브러리를 사용하여 상태를 관리하는 케이스도 많이 있는 것 같은데 상태관리할때 어떤 라이브러리를 선호하시나요?

- 간단하고 작은 프로젝트나 특정 상황에서는 Context API가 사용하기 매우 편리하지만, 복잡한 상태 관리가 필요한 대규모 프로젝트에서는 Recoil이나 Zustand 같은 라이브러리가 아직은 더 적합하다고 생각하는데 Context API가 커버할수 있는 기능이 계속 업데이트 되고 있는걸로 봤을때 추후에 다른 라이브러리의 역할을 대체할수 있을까요?
